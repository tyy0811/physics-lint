## Task 4 — PH-BC-001 (boundary-condition violation) F3 pin

**Rule semantic target.** PH-BC-001 emits a boundary-condition violation metric:
how much a model's prediction violates the prescribed BC on the domain boundary.
The rule's docstring / comparable quantity is the BC-residual norm in the trace
space (per complete-v1.0 plan Task 4 Function 1 anchor: Evans §5.5 Thm 1).

**PDEBench bRMSE definition.** PDEBench main paper §3.3 "Metrics": `bRMSE` is
defined as "RMSE on boundary (whether boundary condition can be learned)." This
measures the baseline model's prediction-RMSE at boundary grid points. Formal
definition equals

    bRMSE = sqrt( (1/|∂Ω_disc|) · Σ_{x ∈ ∂Ω_disc} (u_pred(x) − u_true(x))² )

where `∂Ω_disc` is the discrete boundary point set.

**Semantic-compatibility check per §6.4 Category 8.**
- PH-BC-001 measures `u_pred − u_BC_prescribed` (the rule knows the prescribed
  BC value and compares the prediction to it).
- PDEBench bRMSE measures `u_pred − u_true` at boundary points, where `u_true` is
  the high-fidelity numerical reference (not the prescribed BC as such).
- **Equivalence when the reference simulator satisfies the BC exactly.** For the
  PDEBench datasets generated by finite-volume / upwind / MUSCL solvers with
  strongly imposed Dirichlet or Neumann BCs, the reference `u_true(x)|_{∂Ω}` is
  numerically identical to the prescribed BC (within solver roundoff), so
  `u_pred − u_true|_{∂Ω} = u_pred − u_BC_prescribed|_{∂Ω}` and PDEBench's bRMSE is
  semantically equivalent to PH-BC-001's emitted quantity.
- **Caveat.** For PDEBench datasets whose BC is implicit or generated by the
  solver (periodic BC on advection; random initial field on compressible NS with
  out-going BC), the equivalence still holds within numerical precision — the
  boundary value used by the reference is the PDEBench ground-truth, and the
  linter compares prediction to that same ground-truth boundary value. For
  *Neumann* BCs (e.g., 2D diffusion-reaction's no-flow `D∂_x u = 0` on the
  boundary), the comparison is value-not-normal-derivative — a strict physics-lint
  trace-theorem anchor would want the normal derivative, but PDEBench's bRMSE
  reports the value, so physics-lint's PH-BC-001 Dirichlet fixture maps directly
  while a Neumann fixture would need explicit handling. Complete-v1.0 plan Task 4
  §12 scope: V1 covers Dirichlet / Neumann / periodic BC fixtures; Neumann
  fixture pinned separately.

**Pinned rows (F3-promotable).** PDEBench supplement Tables 5–7 report bRMSE
values per PDE / parameter / baseline model. Specific rows the Task 4 fixture
can target:

1. **Diffusion-sorption** (PDEBench Table 5, "Summary of the baseline models'
   performance for different evaluation metrics"): bRMSE for U-Net = 6.1 × 10⁻³,
   FNO = 2.0 × 10⁻³, PINN = 1.4 × 10⁻¹. PDE: 1D diffusion-sorption (supplement
   §D.3). Dirichlet / Neumann mixed BC on the 1D domain.
2. **2D diffusion-reaction** (PDEBench Table 5): bRMSE for U-Net = 7.8 × 10⁻²,
   FNO = 2.7 × 10⁻², PINN = 2.2 × 10⁻¹. PDE: 2D Fitzhugh-Nagumo activator-inhibitor
   system (supplement §D.9); no-flow Neumann BC `D∂_x u = 0` on `(-1,1)²`.
3. **Shallow-water equation** (PDEBench Table 5): bRMSE for U-Net = 4.2 × 10⁻³,
   FNO = 1.4 × 10⁻³, PINN = 1.5 × 10⁻¹. PDE: 2D shallow-water.
4. **1D Advection** (PDEBench Table 6): bRMSE for U-Net = 3.8 × 10⁻²
   at β = 0.1 (supplement §D.7); FNO = 4.9 × 10⁻³; PINN = 7.8 × 10⁻¹.
   Periodic BC.

**Disposition: F3-present, Task 4 pins Diffusion-sorption + 2D diffusion-reaction
+ Advection rows.**

- Diffusion-sorption (Dirichlet-dominant) gives the Dirichlet fixture's
  reproduction target.
- 2D diffusion-reaction (Neumann no-flow) gives the Neumann fixture's
  reproduction target. Complete-v1.0 plan Task 4 Pre-execution audit
  step (c) is flagged for the Dirichlet-vs-Neumann distinction; the
  reproduction setup uses two PDE targets, not one.
- Advection (periodic) gives the periodic fixture's reproduction target.

Comparison target for each: the U-Net and FNO bRMSE values (the PINN column is
noisier per PDEBench's own commentary and is not a stable reproduction target).
CITATION.md Task 4 Function 3 pins these rows; reproduction tolerance defined as
"physics-lint's emitted bRMSE on the same dataset row must land within ±2× of the
reported U-Net / FNO numerical values" (the 2× envelope accommodates run-to-run
seed variance reported in Takamoto et al. §3.5).

## Task 8 — PH-CON-001 (heat mass conservation) F3 pin

**Rule semantic target.** PH-CON-001 emits a mass-conservation violation metric
on heat-equation-type PDEs: how much ∫_Ω u(t,x) dΩ deviates from its initial
value over the time window. Complete-v1.0 plan Task 8 Function 1 anchor: Evans
§2.3 + Dafermos Chapter I.

**PDEBench check (first pass).** PDEBench does *not* include a pure heat equation
dataset. The closest candidates are:
- **1D Diffusion-Reaction** (supplement §D.2): `∂_t u − ν ∂_{xx} u − ρ u(1-u) = 0`.
  The reaction term `ρ u(1-u)` breaks mass conservation: the total mass is not
  constant in time. Semantically incompatible with PH-CON-001's emitted quantity
  (conservation-of-mass-on-heat-equation).
- **2D Diffusion-Reaction** (supplement §D.9): activator-inhibitor Fitzhugh-
  Nagumo; reaction terms break conservation of the individual variables, same
  incompatibility.
- **Diffusion-sorption** (supplement §D.3): boundary-sorption model with mass
  exchange across the boundary; mass is *not* conserved in the pure-bulk sense
  physics-lint PH-CON-001 checks. Semantically incompatible.

**PDEBench disposition: no directly-comparable row exists for Task 8.** Per §1.2
calibration-vs-reproduction distinction: PDEBench's cRMSE is defined for non-
mass-conserving PDEs and does not provide a clean reproduction target for heat-
equation mass conservation. PDEBench demoted to Supplementary calibration
context for Task 8 CITATION.md.

**Hansen ProbConserv check.** Hansen §3.1 ("The Diffusion Equation: k(u) = k") is
the "easy" case of the Generalized Porous Medium Equation (GPME). Hansen
explicitly states: "The diffusion equation is also known as the heat equation,
where in that application the PDE parameter k denotes the conductivity and the
total conserved quantity denotes the energy. In our empirical evaluations, we
use the diffusion equation notation, and refer to the conserved quantity as the
mass." Hansen's conservation-error (CE) metric is defined as `CE(μ) =
(Gμ − b)_{t_j}` where G is the integral-conservation operator and b is the
conservation constraint. The `L²` norm of the CE is `CE2 = ‖Gμ − b‖²₂`. Hansen
Table 1 reports this metric for the easy-case diffusion equation at t = 0.5,
k ∈ [1, 5] train / k = 1 test, over n_test = 50 runs:

| Model          | CE × 10⁻³    | LL        | MSE × 10⁻⁴ |
|----------------|--------------|-----------|------------|
| ANP            | 4.68 (0.10)  | 2.72 (0.02) | 1.71 (0.41) |
| SoftC-ANP      | 3.47 (0.17)  | 2.40 (0.02) | 2.24 (0.78) |
| HardC-ANP      | 0 (0.00)     | 3.08 (0.04) | 1.37 (0.33) |
| ProbConserv-ANP| 0 (0.00)     | 2.74 (0.02) | 1.55 (0.33) |

(Mean and standard error over 50 runs; numbers taken verbatim from Hansen 2024
Physica D Table 1 / arXiv:2302.11002 Table 1.)

**Semantic-compatibility check per §6.4 Category 8.**

Hansen CE measures `(Gμ − b)|_{t_j}` — the deviation of the model's predicted
mean μ from exact conservation at time index t_j. For the diffusion equation
with zero-flux boundaries and a random-field IC centered on zero, the true total
mass U(t) = ∫_Ω u(t,x) dΩ is zero at all t, so the CE metric at test-time
t = 0.5 equals the absolute deviation of ∫_Ω u_pred(0.5, x) dΩ from zero.
Physics-lint PH-CON-001 measures `|∫_Ω u_pred(t,x) dΩ − ∫_Ω u_pred(0,x) dΩ|` —
the deviation of current total mass from initial total mass. For Hansen's setup
both quantities equal zero by construction (random-field IC has zero mean),
so the two metrics are semantically equivalent.

**Disposition: F3-present, Task 8 pins Hansen Table 1 ANP row.**

Reproduction target: Task 8's CITATION.md Function 3 reports physics-lint's
emitted conservation-violation metric on Hansen's pre-trained ANP predictions
for the diffusion-equation test at t = 0.5, k = 1. Target range: ANP reference
value `4.68 × 10⁻³` within the published standard error of `0.10`; tolerance
for physics-lint's reproduction `±3σ = 0.30 × 10⁻³` absolute, corresponding to a
~6.5 % envelope. The ANP predictions are obtainable from the ProbConserv repo
(`github.com/amazon-science/probconserv`, MIT license) as pre-trained
checkpoint outputs; physics-lint's reproduction layer loads those predictions
and computes the conservation metric — no ML training on physics-lint's side.

**Alternative F3 row for same task (backup pin).** Hansen Table 1 HardC-ANP
row: CE = 0 exactly. This is a harder-to-meaningfully-compare row because the
conservation metric is enforced to zero by construction — physics-lint's
emitted metric must also land numerically at zero (within float tolerance) on
the HardC-ANP predictions. Useful as a zero-check sanity layer, not as a
strength-of-reproduction claim.

## Task 9 — PH-CON-002 (wave energy conservation) F3 pin

**Rule semantic target.** PH-CON-002 emits a wave-equation energy-conservation
violation metric: how much E(t) = ½ ∫_Ω (u_t² + c² |∇u|²) dΩ deviates from
E(0) over the time window. Complete-v1.0 plan Task 9 Function 1 anchor: Evans
§2.4.3 energy identity + Strauss §2.2 + Hairer-Lubich-Wanner Ch IX.

**PDEBench check.** PDEBench does not include a standalone wave equation
dataset. The closest candidates are:
- **Shallow-water equation** (supplement §D.6): `∂_t h + ∇·(hv) = 0`,
  `∂_t (hv) + ∇·(hvv^T + ½gh²I) = 0`. Hyperbolic, but PDEBench reports only
  mass-conservation cRMSE, not energy-conservation. The shallow-water energy
  is a different functional form from the wave-equation energy; semantically
  incompatible with PH-CON-002's emitted quantity.
- **Compressible Navier-Stokes** (supplement §D.5): conservative form with
  mass, momentum, and energy conservation laws. PDEBench's cRMSE for
  compressible NS is on the mass row, not the energy row; the paper does
  not break out energy-conservation metrics. Semantically incompatible with
  PH-CON-002.

**PDEBench disposition: no directly-comparable row exists for Task 9.**
PDEBench demoted to Supplementary calibration context; PH-CON-002 CITATION.md
notes "PDEBench reports mass-conservation cRMSE on shallow-water and
compressible-NS; wave-equation energy conservation is not a PDEBench metric."

**Hansen ProbConserv check.** Hansen's diffusion / PME / Stefan GPME family
is a parabolic / degenerate-parabolic family; wave equation is not in scope
(the CE metric is framed for first-order-in-time integral conservation laws,
not second-order-in-time energy functionals). Hansen demoted to Supplementary
calibration context for Task 9.

**Disposition: F3-ABSENT with justification.** Task 9 CITATION.md Function 3
subsection states: "No directly-comparable published numerical baseline exists
for wave-equation energy-conservation error on the rule's emitted quantity
(E(t)/E(0) ratio on leapfrog-integrated periodic wave equation). PDEBench's
cRMSE is a mass-conservation metric on different PDEs; Hansen's CE is an
integral-conservation metric on first-order-in-time PDEs. Neither provides a
reproduction target for a second-order-in-time wave-energy functional.
Supplementary calibration context: PDEBench shallow-water cRMSE (mass, not
energy); Hansen ProbConserv CE (first-order, not second-order energy)." Per
complete-v1.0 plan §1.2, this is a structural absence, not a validation gap.
